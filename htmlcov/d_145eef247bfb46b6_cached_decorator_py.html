6class cached(object): 

7 """Decorator that caches a function's or class method's return value for cache_time_seconds. 

8 If called later before cache_time_seconds passed with the same arguments, the cached value 

9 is returned, and not re-evaluated. 

10 Example of usage: 

11 @cached(0.1) 

12 def function_to_cash_for_one_tenth_of_second(): 

13 

14 ! For object method it should be the only decorator applied because it saves ref to decorated function 

15 and if it changed by other decorator it would treat method call as function call, and 

16 in this case if the class is not singleton it would cache different instances with different hashes. 

17 And would not use the object attributes in hash. 

18 

19 All instances of the same class would be the same for cache decorator if they has the same 

20 attributes. 

21 So the object do not need to be singleton to be cached. 

22 For example: 

23 obj = MyObject(1) 

24 obj.my_func(2) 

25 obj.my_func(2) # cached 

26 obj2 = MyObject(1) 

27 obj2.my_func(2) # also cached 

28 obj3 = MyObject(2) 

29 obj3.my_func(2) # not cached, if MyObject __init__ save something different in self 

30 # for different __init__ parameter 

31 

32 It compares hash of all function's arguments (str representation), 

33 and __dict__ of it's first argument if it is an object with one of it's methods the same 

34 as decorated function. 

35 So if you change something in the object (any self.<attribute>), the decorator also 

36 would see that. 

37 !!! Str representations for function arguments should be distinguishable. 

38 That's true at list for python's list & dict. 

39 

40 Decorator instance is one per decorated function/method, so we do not need to take into account 

41 function and object name (if this is object method), because for other functions/methods it will 

42 be other decorator instance. 

43 """ 

44

48 :param evaluate_on_day_change: re-evaluate if current day is not the same as cached value

63 and '__func__' in dir(getattr(args[0], func.__name__)) \
